## 一、结论

V2 已于 2026-07-21 发布到 GitHub 仓库 `main`，网页、SQL、发布说明和本报告与本地校验版本完全一致；GitHub Pages 最终构建与部署成功。别人现在打开公开网址会进入整改后的 V2。

基础学习、单词、复习、课程入口和练习已经上线；最新 Supabase SQL 与正式跳转 URL 已配置。完整字母与重点拼读规则已接入公开母语者真人录音。两个 Supabase Edge Function 已部署：AI 老师被固定为 Gemini 免费优先，未配置免费 key 时只使用离线老师；付费 TTS 默认关闭。完整账号云进度还需要双账号、跨设备实际验收；60 集视频只有教材知识摘要，不是老师讲课的完整逐字讲义。

## 二、账号和跨设备进度是否独立

### 整改后的设计

| 使用方式 | 数据结果 |
|---|---|
| 不登录（访客） | 只保存在当前浏览器，和登录账号分开；换设备不会同步。 |
| 同一个账号登录电脑、iPhone、iPad | 合并同一账号的生词、复习、课程完成、DW/视频完成、闯关、今日计划和已掌握词。 |
| 不同账号登录同一台设备 | 使用不同的本机存储命名空间，并读取数据库中各自的 `user_id` 行，进度彼此独立。 |
| 不同账号登录不同设备 | 数据库按 `user_id` 分行，进度彼此独立。 |

数据库脚本已启用并强制 Row Level Security，只允许登录用户读取、插入和更新 `auth.uid() = user_id` 的行。浏览器端也不再让所有账号共用一套无账号前缀的学习状态。

为了不丢失旧版数据，旧版浏览器里的未分账号数据只允许第一个登录账号认领一次，并在首次同步时与该账号已有云数据做一次无损合并；后续账号不会再自动合并这份旧数据。

### 必须说明的验证边界

代码和 SQL 策略已经完成静态核对；生产接口检查已从旧版的“`progress` 字段不存在”变为匿名访问 401 受保护，符合新 SQL 添加字段并撤销匿名表权限后的预期结果。仍需按发布说明用两个真实邮箱交叉登录验收，才能把生产环境标记为“已验证独立”。

当前 GitHub 线上代码已经使用账号作用域存储和完整进度同步，SQL 与 Auth URL 也已配置；但在双账号尚未实测前，**生产环境仍不能标记为“已验证独立”**。

## 三、本轮性能与稳定性整改

- 单词库由一次渲染全部卡片改为每批 24 张，搜索增加短延迟合并输入，减少手机滚动和频繁重绘。
- 生词列表按批显示，新加生词仍置顶；不再首次打开就自动塞入 240 个课程词。
- Bilibili 元数据增加 7 天缓存和 6 秒超时，并内置已核实的 60 集时长；网络受限时页面仍可工作。
- 不再运行时搜索未知的词典录音；发音课改用已核实文件名的固定 Wikimedia 母语者录音，一次点击只加载一段音频，并为每段录音显示来源与许可。
- 全站语音默认零费用：固定字母/规则优先使用公开真人录音，任意文本和缺失录音使用设备德语语音；付费 TTS 后端默认拒绝请求，只有管理员显式开启才可能产生费用。
- 单词卡点击改为容器级统一处理，不再在每批卡片重画后重复创建多组监听器；搜索文本建立内存缓存，切换“今天学习”不再触发重复云同步和首页重画。
- 在线老师保留最近 8 条对话上下文，提交期间锁定按钮并使切换场景时的旧请求失效；后端缓存 10 分钟内的完全相同请求，减少重复额度消耗。
- 登录状态切换增加重复事件抑制，避免页面初始化时同一账号重复同步和重复重绘。
- 登录入口移到页面最上方，并始终明确显示当前是访客进度还是某个邮箱账号的独立云进度；生词区不再重复放置第二套登录框。
- 学习状态写入改为账号作用域，并用 700ms 合并云同步请求。
- 修复“删除生词或取消完成后又被云端合并回来”的问题：已有时间信息时以最近修改的一端为准，只有旧数据首次迁移时才做无损合并。
- 修复同步进行中再次修改时可能漏掉最后一次写入的问题；进行中的修改会排队，在当前请求结束后再次同步。
- 切换或退出账号时会取消旧账号尚未开始的同步；已发出的旧请求也会在写入本机状态前再次核对账号，避免跨账号污染。
- JavaScript 已通过语法检查；HTML 中未发现损坏的 UTF-8 替换字符。

本轮还用 Windows Edge 的 390 × 844 手机视口做了页面运行测试：初始化无 JavaScript 页面错误，整页无横向溢出，单词库首批为 24 张，96 条已渲染例句全部常驻可见，快速加入 `Vokabeln` 能自动补义并置顶，上一课/下一课切换正常。浏览器内模拟账号 A/B 后，两套作用域存储分别保持 A/B 数据，没有互相覆盖。

## 四、60 集课程内容核对

已通过该 Bilibili 合集的公开接口核实：共 60 集，总时长 190,170 秒，约 52.8 小时。逐集字幕接口检查结果为 60 集全部 0 条公开字幕轨道。

目前页面有 60 集目录、每集主题、语法和 4 个核心词，共 240 个摘要词；这属于教材结构化摘要。只有前几课的一条龙讲解做了较深的人工内容包，后续课程仍会使用通用生成的例句和练习。因此用户指出“视频里老师讲的很多没有写出来”是正确的。

在没有字幕、讲义或逐集人工听记的情况下，不能可靠补出老师讲的每个例子、板书、纠错和补充说明。本轮已经在课程卡上加入“教材知识摘要、非逐字稿、无公开字幕”的状态标记，避免再次误导。

要真正补齐，建议按 60 集逐集处理：合法字幕/讲义 → 逐段核对 → 知识点 → 例句 → 生词 → 练习 → 人工复核。52.8 小时课程不应靠分集标题自动猜测。
